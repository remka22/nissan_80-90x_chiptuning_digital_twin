# ЭБУ-двойник M30 (SIL — software-in-the-loop)

Виртуальный стенд: гоняем **реальный код прошивки** M30 на эмуляторе процессора
**HD6303**, подаём входы (обороты, MAF/ДАД, температура, нагрузка) и смотрим,
**по каким веткам пошёл код и что он насчитал** — без машины и без логов с авто.

Это **не** симулятор мотора. Двойник считает то, что ЭБУ **командует**
(длительность впрыска, угол, нагрузку), а не то, что мотор из этого сделает.
Реальный AFR/мощность — по-прежнему только на машине. Зато логику патчей
(speed-density, две карты, отсечка) и пороги переключения режимов проверяем 100% тут.

## Что умеет
- **Одиночный прогон** рутины с заданными входами → значения наблюдаемых переменных,
  дошла ли до RTS, не упала ли, что записала в порты.
- **Свип** одного входа (например, поднять обороты 0→6000) → график: видно,
  на каком значении щёлкает флаг/переключается ветка.

## Запуск (Docker)
```
cd ecu_twin
docker compose up --build
```
Открыть в браузере: http://localhost:8000

Прошивка монтируется из `../m30/infiniti90m30at (1).bin` (см. docker-compose.yml).

## Запуск без Docker (локально)
```
cd ecu_twin
pip install -r requirements.txt
ECU_ROM="../m30/infiniti90m30at (1).bin" uvicorn api:app --app-dir backend --port 8000
```

## Структура
```
backend/
  emulator/
    cpu6303.py   — ядро процессора HD6303 (инструкции, флаги, 6303-расширения)
    machine.py   — память 64К, ROM, перехват железа (ADC $1006/$1007, порт $2000),
                   прогон рутины до RTS, свипы
    symbols.py   — карта входов/наблюдаемых/рутин (из нашего реверса)
  api.py         — FastAPI: /api/meta, /api/run, /api/sweep
frontend/
  index.html, app.js, style.css — браузерный UI + графики (Plotly)
```

## Точки входа (рутины) сейчас
- `maf_to_load` @ 0x899B — обработка MAF: `$1408 → VQ(0xFA70) → $1577`
- `tp_filter`   @ 0x8A6A — фильтр Tp: `$1413 → $1482`
- `load_calc`   @ 0x89D4 — расчёт нагрузки `$1413`
