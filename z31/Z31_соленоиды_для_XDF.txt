==============================================================================
 Z31 (VG30ET турбо) — УПРАВЛЯЕМЫЕ ВЫХОДЫ и КЛАПАНЫ для выноса в XDF
 Драйвер: FUN_9ff1 (cpu 0x9FF1). Выходной байт-порт: $00B7.
 Метод: декомпиляция Ghidra + кластеризация «механизм + только его данные».
 BASE Z31 = 0x4000 -> адрес XDF (file) = CPU - 0x4000. Калибровки CPU 0xBFxx = file 0x7Fxx.
 [ФАКТ] = из кода. [ГИП] = опознание по профилю/данным (точный бит->вывод = распиновка).
==============================================================================

 ВСЕГО на $00B7 — 6 управляемых бит-выходов (+ парные защёлки 0x10/0x20). [ФАКТ]
 Из них вакуумных соленоидов-клапанов — три (EGR / буст / абсорбер).


==============================================================================
 КЛАПАН 1 — EGR  =  бит $00B7.0x04            [ФАКТ структура; ГИП имя — высокая увер.]
==============================================================================
 Поведение [ФАКТ]: ступенчатое включение по оборотам + потолок нагрузки + выдержка.
   обороты(DAT_008c) >= порог, ступени; И нагрузка(DAT_00b8) < лимит; И DAT_00c3==0.
 Почему EGR [ГИП, сильная]: единственный выход со СВОИМ замкнутым банком калибровок
   (ступени оборотов + значения + лимит нагрузки) — классический профиль EGR;
   лямбда НЕ участвует (верно для EGR). Флаг состояния — DAT_00c3 (общий только с 0x02).

 СОБСТВЕННЫЕ ДАННЫЕ (в XDF, адрес=file):
   0x7FEA = 10   ступень оборотов 1 (нижний порог открытия)
   0x7FEB = 26   ступень оборотов 2
   0x7FEC = 36   ступень оборотов 3
   0x7FED = 48   значение для ступени (с 0x7FEB)
   0x7FEE = 80   значение для ступени (с 0x7FEC)
   0x7FEF = 8    значение
   0x7FF0 = 4    значение
   0x7FC0 = 48   базовое значение
   0x7FC8 = 12   базовое значение
   0x7FC9 = 48   ЛИМИТ НАГРУЗКИ (выше — EGR выкл)
   0x7FF1 = ...  доп. (запись состояния DAT_00c3)
 Переменные-входы (RAM, не в XDF): DAT_008c=обороты, DAT_00b8=нагрузка,
   DAT_007d=расчётная дельта, DAT_00c3=флаг состояния EGR.


==============================================================================
 КЛАПАН 2 — БУСТ (соленоид буст-контроля)  =  бит $00B7.0x40   [ФАКТ структура; ГИП имя]
==============================================================================
 Поведение [ФАКТ]: включён КОНФИГОМ (турбо), управляется импульсом/таймером.
   разрешение: DAT_bfd3 & 0x20 ; условие: DAT_001f & 0x20 ; импульс: DAT_0100/DAT_0101.
 Почему буст [ГИП]: конфиг-зависимый (турбо-опция) + ШИМ-импульс — профиль соленоида
   управления давлением наддува. На атмо-вариантах конфиг-бит был бы снят.

 СОБСТВЕННЫЕ ДАННЫЕ (в XDF, адрес=file):
   0x7FD3 = 181 (0xB5)  КОНФИГ; бит 0x20 = разрешение буст-соленоида
 Переменные-входы (RAM): DAT_0019, DAT_001a, DAT_0053, DAT_001f, таймер DAT_0100/0101.


==============================================================================
 КЛАПАН 3 — АБСОРБЕР (canister purge)  =  бит $00B7.0x02   [ФАКТ структура; ГИП имя]
==============================================================================
 Поведение [ФАКТ]: режим-зависимый (ветка TRCS&1≠0), гейт по расходу и флагам.
   условие: DAT_001f & 8 ; DAT_00a3 ; расход DAT_000c vs порог DAT_bf85(=0) ; FUN_b096.
 Почему абсорбер [ГИП]: простой режимный вкл-выкл по расходу/режиму (не по оборотам-окну
   и не конфиг-турбо) — остаётся продувка адсорбера. Делит флаг DAT_00c3 с EGR (emission).
 Альтернатива: им мог бы быть и бит 0x08 (тоже режимный, дебаунс DAT_0037→64,
   своя калибровка 0x7FC1). Какой из 0x02/0x08 именно абсорбер — без распиновки не разнести.

 СОБСТВЕННЫЕ ДАННЫЕ (в XDF, адрес=file):
   0x7FC2 = ...  калибровка ветки бита 0x02
   0x7FC1 = ...  калибровка ветки бита 0x08 (второй режимный кандидат)
 Переменные-входы (RAM): DAT_000c=расход, DAT_001f, DAT_00a3, DAT_0037=дебаунс, TRCS=режим.


==============================================================================
 ОСТАЛЬНЫЕ выходы $00B7 — НЕ вакуумные клапаны                      [ФАКТ поведение]
==============================================================================
 бит 0x01 — порог оборотов: ВКЛ при DAT_0048>79, ВЫКЛ при <16 (гистерезис).
            Профиль РЕЛЕ — вероятно реле бензонасоса (вкл при раскрутке). [ГИП]
            Прим.: вторая стадия «насос напрямую по температуре» в этом бите НЕ видна —
            значит отдельный выход/железо.
 бит 0x80 — быстрый ТАЙМЕР/фазовый: ВКЛ когда счётчик DAT_0028==0; свой банк 0x7780-0x7791
            (file), счётчики 0x20-0x2E, serial TX. НЕ клапан. [ФАКТ]


==============================================================================
 ЧТО ТОЧНО ПОНЯЛИ (итог)
==============================================================================
 [ФАКТ] $00B7 — выходной порт; драйвер FUN_9ff1; 6 управляемых бит.
 [ФАКТ] бит 0x04 — со СВОИМ банком калибровок 0x7FEA-0x7FF1 (ступени оборотов+лимит нагрузки).
 [ФАКТ] бит 0x40 — конфиг-зависимый (0x7FD3 bit0x20) + импульс.
 [ФАКТ] лямбда (O2) НЕ участвует ни в одном из 6 — это клапаны/реле/таймер, не топл. коррекция.
 [ГИП, сильная] EGR=0x04, буст=0x40, абсорбер=0x02(или 0x08), реле насоса=0x01, таймер=0x80.
 Точный бит->вывод разъёма->клапан окончательно подтверждается РАСПИНОВКОЙ ЭБУ.
 (Аналогичный разбор соленоидов M30 — см. ../m30/M30_соленоиды_EGR_для_XDF.txt)
==============================================================================



==============================================================================
 ПЕРЕСМОТР: РХХ (регулятор ХХ) = ШИМ (моментный). Подтверждено железом + машиной.
==============================================================================
 Проверка таймера output-compare (OCR 0x000B/0x000C): у Z31 записей в OCR = 3
   -> используется АППАРАТНЫЙ ШИМ -> РХХ Z31 = ШИМ (моментный соленоид по duty).

 ВАЖНО: РХХ Z31 — это ОТДЕЛЬНЫЙ аппаратный ШИМ-выход (через OCR-таймер),
   а НЕ один из бит $00B7. Значит 6 бит $00B7 — это клапаны/реле/таймер,
   а регулятор ХХ идёт мимо них (своя ШИМ-линия). DAT_0100/0101 — программные
   таймеры реле/импульсов, не путать с idle-ШИМ.

 Контраст с M30: там РХХ — ШАГОВЫЙ (4 GPIO-бита, OCR не используется, 0 записей).
   Z31 (моментный ШИМ) vs M30 (шаговый 4 шага) — разная механика РХХ, что и
   объясняет, почему у M30 на actuator-байтах лишние 4 «выхода» (это 4 фазы шаговика),
   а у Z31 их нет. См. ../m30/M30_соленоиды_EGR_для_XDF.txt

 УТОЧНЁННЫЙ СОСТАВ Z31:
   $00B7: EGR(0x04), буст(0x40), абсорбер(0x02/0x08), реле насоса(0x01), таймер(0x80)
   РХХ ШИМ: отдельный аппаратный выход (OCR-таймер), вне $00B7
==============================================================================
