 Z31 — ПОЛНЫЙ XDF-ИНВЕНТАРЬ (максимум данных для своей XDF, чисто статически)
 Получено перечислением ВСЕХ таблиц, которые код реально читает (инструкции LDX#),
 + классификацией по подпрограмме поиска (1D/2D/порог) + опознанием входной
 RAM-переменной (чем индексируется). Карта памяти: CPU = file + 0x4000.
==============================================================================
 Код читает 43 таблицы. 17 из них уже описаны в готовом XDF (помечены [XDF]).
 26 — НОВЫЕ (XDF их не знал). Все они ниже, со структурой для добавления в XDF.

 Тип ячеек везде 8-бит, кроме VQ (16-бит). Размер большинства кривых/осей = 16x1.
 Подпрограммы: $80D2/$80DD = 1D-поиск, $815D/$8021 = 2D, $7096 = сравнение с порогом.

 СЛОВАРЬ ВХОДОВ (чем индексируются таблицы):
   0x0F/0x10 = ОБОРОТЫ (RPM)          [ДОКАЗАНО]
   0x20/0x21 = НАГРУЗКА (Tp)          [ДОКАЗАНО]
   0x00FF    = темп. ОЖ ; 0x007E/0x13 = темп. воздуха   [ДОКАЗАНО]
   0x48, 0x8C = индексы, ВЫЧИСЛЕННЫЕ ИЗ RPM            [ДОКАЗАНО]
   0x8A      = темп-индекс (вероятно воздух)           [ГИПОТЕЗА]
   0x0C      = значение датчика с АЦП                  [ДОКАЗАНО (читается из АЦП)]
   0xC0      = флаг/статус ; 0x1038 = вычисляемый параметр
   0xBEE0    = БАЙТ-ОПЦИЯ (селектор вариантов)         [ДОКАЗАНО]

 ОТДЕЛЬНО: БЛОКИ СКАЛЯРОВ-ПОРОГОВ (не читаются как таблица, сравниваются побайтно)
==============================================================================
 0x7780-0x77A7, 0x77E0-0x77EC, 0x7900-0x790A, часть хвоста 0x7F70-0x7FE0 —
 это отдельные СКАЛЯРЫ (8-бит). Каждый — кандидат в XDF как Constant. Часть с
 физикой: пороги темп. воздуха (X-50 = °C), диапазон «обрыв/КЗ» (низ/верх),
 счётчики дебаунса (малые целые). Конфиг-байты — блок 0x7ED9 (вкл. опция 0x7EE0).


==============================================================================
 ИТОГ ДЛЯ XDF
==============================================================================
 Полностью готово к записи (структура известна у ВСЕХ):
   - 5 вариантных копий (альт. VQ + 4 альт. оси) — со scale как у их аналогов;
   - 2 оси самообучения;
   - ~13 1D-кривых коррекции (16x1, 8-бит): 4 «vs обороты», 5 «vs температура»,
     остальные — вход не опознан, но тип/размер известны;
   - 4 пороговые таблицы;
   - 2 неклассифицированные (структуру внести можно);
   - россыпь скаляров-порогов (отдельными Constant).
 Это +26 определений к 17 из готового XDF = ~43 элемента калибровки на Z31.
 Где роль = [ГИПОТЕЗА], имя в XDF пишем рабочее (Correction Curve vs RPM/Temp,
 Threshold Table, …) — XDF при этом корректно ДЕКОДИРУЕТ (структура верна).

==============================================================================
 ДОПОЛНЕНИЕ: ВЫВОДЫ ПО САМИМ ЗНАЧЕНИЯМ (доказано числами)
==============================================================================

 ВЫВОД 1. Вариант = РАЗНЫЙ ВПУСК/РАСХОДОМЕР (не «копия»).            [ДОКАЗАНО числами]
   Сравнение осей основная vs альтернативная:
     - RPM-оси и timing-оси у вариантов ИДЕНТИЧНЫ;
     - ось НАГРУЗКИ топлива РАЗНАЯ: осн 2,3,4,…,16 ; альт 2,2.5,3,3.5,4,…,14
       (мельче на малой нагрузке, ниже потолок).
   => второй вариант (альт VQ 0x79B0 + альт ось нагрузки 0x7AB0) — под ДРУГОЙ
   расходомер/впуск, а не другой мотор.

 ВЫВОД 2. ЧАСТЬ «таблиц» НЕАКТИВНА в этой прошивке (важно для XDF).  [ДОКАЗАНО числами]
   0x7960 = всё 255  -> отключена/кламп
   0x7970 = всё 255  -> отключена/кламп
   0x7B90 = всё 0     -> пустая (не используется)
   0x79A0 = всё 60    -> КОНСТАНТА (не кривая)
   0x7BC0 = всё 64    -> КОНСТАНТА (0x40, вероятно опорная «1.0»)
   В XDF их пишем как плоские/скаляры, интерпретировать как кривые не нужно.

 ВЫВОД 3. Темп-кривые = ПРОГРЕВОЧНОЕ ОБОГАЩЕНИЕ.                     [ДОКАЗАНО формой]
   0x7E00: 59->6 ; 0x7E10: 77->0 ; 0x7E60: 100->56 — спад «холодно->горячо».
   Подтверждает «vs температура» = обогащение/коррекция по прогреву.

 ВЫВОД 4. RPM-кривые = коррекции на ВЫСОКИХ оборотах.               [ДОКАЗАНО формой]
   0x7ED0: 0 на низах -> 60 на верхах (работает только сверху)
   0x7EA0: 10->35 (рост) ; 0x7F60: 15->25 (ступень вверх).

 ИТОГО уточнение для XDF: из 26 новых — 5 неактивных (флаги/константы),
 ~8 активных кривых с понятной ролью (прогрев / высокие обороты),
 5 вариантных копий (другой впуск), 2 оси самообучения, 4 порога, 2 неясных.

 ДОПОЛНЕНИЕ: ИМЕНА ИЗ RB20-XDF (закрывают наши безымянные) [сверено по значениям]

 Прошли RB20det.xdf (R32) по адресам, декодировали те же адреса в Z31 —
 значения осмысленные, значит имя применимо и к нам:

   0x76B0  Base Idle RPM (целевые ХХ vs темп.)  -> Z31 ~788 об/мин (≈ заводские 750)
   0x79E3  High Octane Knock Coeff.             -> 24..182
   0x7FE0  Low Octane Knock Coeff.              -> 0..6  (ранее «неясная» в инвентаре!)
   0x7B90  Timing retard versus temperature     -> все 0 (таблица есть, выключена)
   0x7F92  O2 Sensor RICH Voltage Limit         -> 0.58 В (X*0.01)
   0x7F93  O2 Sensor LEAN Voltage Limit         -> 0.23 В
   0x7FA4  Rev Limit 2                          -> 5200 об/мин
   0x7FC8  Map Switching Constant               -> 12
   0x7EE0  FLAG: O2 sensor disable
   0x7761  FLAG: Fuel Map Correction Disable
   0x6AD1  ROM ID                               -> данные (5 байт)

 НЕ применимо к Z31 (RB20-специфика, у нас на этих адресах другое/код):
   VTC (0x79F2-0x79FF), 5th-gear/Knock Fuel карты (0x7000/0x7100/0x7200),
   Electric Fan (0x7442-0x7445, 0x7F75/0x7F76), parked rev limiter (0x7FE9).

 ДОПОЛНЕНИЕ: СОЛЕНОИДЫ-КЛАПАНЫ (из кода, драйвер FUN_9ff1, выход $00B7)
==============================================================================
 Что точно поняли по управляемым выходам Z31 (полный разбор — Z31_соленоиды_для_XDF.txt):
   EGR       = бит $00B7.0x04 — свой банк калибровок 0x7FEA-0x7FF1
               (ступени оборотов 10/26/36 + лимит нагрузки 0x7FC9=48)
   БУСТ      = бит $00B7.0x40 — конфиг 0x7FD3 (бит 0x20) + импульс
   АБСОРБЕР  = бит $00B7.0x02 (или 0x08) — режимный, калибровки 0x7FC2 / 0x7FC1
   (не клапаны: 0x01 = реле насоса по оборотам; 0x80 = таймер, банк 0x7780-0x7791)
 Лямбда (O2) ни в одном из выходов не участвует. Точный бит->вывод — распиновка.
