==============================================================================
 РАСШИФРОВКА ПРОШИВКИ 88_Z31turbo32k.bin ПО ОПРЕДЕЛЕНИЯМ Z31.xdf
 Источник: Z31.xdf ("XDF for Nissan 300zx VG30ET 1988-1989 Z31 ECU")
 BinSize=0x8000, BaseOffset=0 -> адреса XDF = смещения в файле напрямую.
 XDF покрывает 902 из 3584 байт зоны данных (~25%). Остальное — ниже, в части 4.

==============================================================================
 ЧАСТЬ 1. РАСШИФРОВАНО (есть в XDF) — с формулами и реальными значениями
==============================================================================

--- ОСНОВНЫЕ КАРТЫ (16x16, оси = обороты x нагрузка TP) ----------------------

0x7C00-0x7CFF  Primary Timing Map        16x16, 8-бит
   Z (ячейка) eq: 1.0 * X   ->  значение = УГОЛ напрямую (градусы).
   Декодировано: 1 .. 50 град.   Оси: X = TP(нагрузка), Y = RPM.
   ПОДТВЕРЖДАЕТ: наша B1 "зажигание, value=реальные углы" — ТОЧНО.

0x7D00-0x7DFF  Primary Fuel Map           16x16, 8-бит  (ОДНИ байты, ДВЕ формулы!)
   - Open Loop   eq: 1881.6/(X+128)  ->  5.75 .. 14.15 ms (впрыск)
   - Closed Loop eq: 1881.6/(X-64)   ->  значения по той же сетке, др. формула
   Это и есть "состав смеси": открытый цикл (без O2) и замкнутый (с O2) —
   ОДНА таблица байт, читаемая двумя уравнениями.
   ПОДТВЕРЖДАЕТ: наша B2/B3 "состав смеси, открытый цикл + ожидаемый двойник".
   Двойник (closed loop) реально есть — по тому же адресу. Сетка 16x16 (не 8!).

--- ОСИ КАРТ (scale-таблицы, по 16 точек) -----------------------------------

0x7AF0-0x7AFF  TP Scale - Fuel       eq: X*0.125   ->  2 .. 16   (TP*8)
0x7B00-0x7B0F  RPM Scale - Fuel      eq: 50*X      ->  400 .. 6000 RPM
0x7B10-0x7B1F  TP Scale - Timing     eq: X*0.125   ->  2 .. 15   (TP ms)
0x7B20-0x7B2F  RPM Scale - Timing    eq: 50*X      ->  400 .. 5600 RPM
   ВАЖНО: у топлива и зажигания ОТДЕЛЬНЫЕ оси (но обе RPM x TP).
   ПОДТВЕРЖДАЕТ: наш "блок осей B4" — это и есть эти scale-таблицы.

--- ТАРИРОВКИ ДАТЧИКОВ -------------------------------------------------------

0x7A30-0x7AAF  VQ Table              64x1, 16-бит  eq: 1.0*X  ->  120 .. 65535
   VQ = тарировка ДМРВ (MAF): напряжение -> расход воздуха.
   ПОДТВЕРЖДАЕТ: наша B5 "ДМРВ/MAF, 16-бит экспонента" — ТОЧНО.

0x7B30-0x7B4F  Water Temp Sensor Conversion   32x1, 8-бит  eq: X-50  -> -50 .. 205 C
   Тарировка датчика темп. ОЖ (термистор): АЦП -> температура.
   ПОДТВЕРЖДАЕТ: наша B6 "ТОЖ-тарировка, термистор, 32 точки" — ТОЧНО.

--- DWELL (накопление катушки) ----------------------------------------------

0x7B50-0x7B5F  Ignition Dwell Duty (Battery Voltage)  16x1  eq: X*100/256 -> 12.5 .. 99.6 %
0x7B60-0x7B7F  Ignition Dwell Duty (RPM)              32x1  eq: 0.33*X    -> 4.95 .. 76.6 %
   ПОПРАВКА к нашему разбору: 0x7B60 мы считали "осью оборотов" — на деле это
   таблица DWELL от оборотов. Промах.

--- ХОЛОДНЫЙ ПУСК / КОРРЕКЦИИ (по 16 точек) ---------------------------------

0x7E20-0x7E2F  Cold Start Enrichment        eq: X      ->  4 .. 87 ms
0x7E70-0x7E7F  Air Flow Limiter             eq: X      ->  все 0 (отключено)
0x7E80-0x7E8F  TTP Min                      eq: X      ->  8 .. 11 ms
0x7E90-0x7E9F  TTP Max                      eq: X      ->  80 .. 136 ms
0x7EB0-0x7EBF  Warm Start Idle Increase     eq: X*50   ->  0 .. 2150 RPM
0x7EC0-0x7ECF  Cold Start Timing Advance    eq: X      ->  0 .. 25 град BTDC
0x7F30-0x7F3F  Cold Start Idle Increase     eq: X*50   ->  0 .. 2150 RPM
   ПОДТВЕРЖДАЕТ: наш раздел D "кривые = коррекции прогрева/обогащения" — ВЕРНО.

--- СКАЛЯРЫ -----------------------------------------------------------------

0x7F2B  Injector K Value           16-бит  eq: X        =  607
0x7F88  Injector Latency           8-бит   eq: X/100    =  0.75 ms
0x7F91  O2 Sensor Feedback Control 8-бит   eq: X-50     =  10 C (порог входа в замкн. цикл)
0x7FA5  Top Speed Limit            8-бит   eq: 1.2427*X =  136.7 MPH (~220 км/ч)
0x7FB4  Rev Limiter                8-бит   eq: 50*X     =  6450 RPM
   ПОДТВЕРЖДАЕТ: наш "хвост 0x7F70-0x7FFF = скаляры" — ВЕРНО (а не "шум").


==============================================================================
 ЧАСТЬ 2. КОНКРЕТНЫЕ ЧИСЛА ЭТОЙ ПРОШИВКИ (новые знания)
==============================================================================
   Отсечка (Rev Limiter):           6450 об/мин
   Ограничитель скорости:           136.7 MPH (~220 км/ч)
   Задержка форсунки (latency):     0.75 мс
   Вход в замкнутый цикл по O2:     от 10 C ОЖ
   Injector K:                      607
   Угол зажигания (карта):          1 .. 50 град
   Доп. угол на холодную:           до +25 град
   Подъём ХХ на пуске:              до +2150 об/мин
   Оси оборотов:                    топливо 400-6000, зажигание 400-5600


==============================================================================
 ЧАСТЬ 3. СВЕРКА НАШИХ ГИПОТЕЗ С XDF (счёт)
==============================================================================
   ПОПАЛИ:
   + Зажигание @0x7C00, значение=градусы (1:1)            -> Primary Timing Map ✔
   + Смесь/топливо @0x7D00, открытый цикл + двойник       -> Fuel Map Open+Closed ✔✔
   + ДМРВ/MAF @0x7A30, 16-бит экспонента                  -> VQ Table ✔
   + ТОЖ @0x7B30, термистор, 32 точки                     -> Water Temp Conv. ✔
   + Оси @0x7AF0-0x7B2F                                   -> TP/RPM Scale ✔
   + Кривые-коррекции (D) = прогрев/обогащение            -> Cold Start*/Idle* ✔
   + Хвост = скаляры (отсечка, скорость, форсунки)        -> ✔

   ПРОМАХИ / ПОПРАВКИ:
   - 0x7B60 считали "осью оборотов" -> это Dwell Duty (RPM).
   - В середине диалога усомнились, что топливо 16x16 (думали 8-широкое) ->
     XDF подтвердил 16x16. Исходная интуиция (обе карты 16x16) была ВЕРНОЙ.


==============================================================================
 ЧАСТЬ 4. ЕСТЬ В ПРОШИВКЕ, НО НЕТ В XDF (не расшифровано)
==============================================================================
 Это реальные данные в дампе, которых XDF не определяет. Где есть наша
 гипотеза — указана. Точное значение требует кода (Ghidra) или более полного XDF.

   0x7200-0x7206  (7 б)    мелкие скаляры/параметры
   0x7780-0x77A7  (40 б)   разрозненные данные
   0x77E0-0x77EB  (12 б)   мелкие данные
   0x7900-0x790F  (16 б)   мелкие данные
   0x79B0-0x7A2F  (128 б)  >>> ВТОРАЯ VQ/MAF-кривая (16-бит экспонента до 65535),
                           аналог 0x7A30, но XDF её НЕ определяет. Сильный кандидат.
   0x7AB0-0x7AEF  (64 б)   возможно ещё scale-таблицы (4x16) перед осями 0x7AF0
   0x7BA0-0x7BBF  (32 б)   индексные ряды (1..80) — оси/лукап
   0x7BF0-0x7BFF  (16 б)   кривая-спад (коррекция)
   0x7E00-0x7E1F  (32 б)   кривые (перед Cold Start Enrichment)
   0x7E30-0x7E6F  (~53 б)  кривые-коррекции
   0x7EA0-0x7EAF  (16 б)   кривая
   0x7ED9-0x7F2A  (82 б)   наш "шумовой" блок 0x7EE0 + кривая 0x7ED0 — неразобрано
   0x7F40-0x7F47, 0x7F50-0x7F57 (по 8 б)  кривые
   0x7F6C-0x7F87  (28 б)   данные перед скалярами форсунок
   0x7F89-0x7FB3  разрозненные   прочие скаляры (не все в XDF)
   0x7FB5-0x7FFF  (75 б)   хвост: прочие скаляры + ВЕКТОРЫ прерываний/reset

 ВНЕ КАЛИБРОВКИ (XDF такое не описывает в принципе):
   0x0000-0x3FFF  пусто (0xFF)
   0x4000-0x71FF  КОД программы ЭБУ
   заливки 0x3F/0xFF/0x00 в зоне данных — заполнители пустых ячеек


==============================================================================
 ИТОГ
==============================================================================
   XDF подтвердил 7 из 7 основных наших выводов (1 промах по dwell, 1 поправка
   по ширине топливной карты). Получены реальные формулы и значения: отсечка
   6450, лимит скорости ~220 км/ч, диапазоны карт, тарировки MAF и ОЖ.
   Нерасшифрованным осталось ~75% байт зоны данных — там, в т.ч., вторая
   MAF-кривая (0x79B0) и набор коррекционных кривых, которых нет в этом XDF.
   Следующий уровень точности по ним — дизассемблер кода 0x4000-0x71FF.
==============================================================================
