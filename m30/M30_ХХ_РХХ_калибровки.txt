==============================================================================
 M30 — КАЛИБРОВКИ РЕГУЛЯТОРА ХОЛОСТОГО ХОДА (РХХ, шаговый) для XDF
 Найдено по коду (Ghidra-декомпиляция + дизассемблер). Адрес = смещение в файле = адрес XDF.
 РХХ M30 = ШАГОВЫЙ (4 фазы по битам $A7/$AB, OCR не используется). Контур замкнутый по оборотам.
 ВНИМАНИЕ: ниже [ПОПРАВКА] — $1006 это НЕ порт шаговика, а регистр команды ADC.
 [ФАКТ] адрес/роль/сырые значения; [scale] множитель из кода; RPM-перевод см. примечание.
==============================================================================


------------------------------------------------------------------------------
 1) ТЕМПЕРАТУРНАЯ КРИВАЯ ПРОГРЕВА  @ 0x7E20  (16 точек)            [ФАКТ]
------------------------------------------------------------------------------
 индекс: $00B1 (температурный индекс, из датчика ОЖ через таблицу 0x7B30)
 сырые:  96 96 81 64 46 32 18 11 9 7 6 6 6 6 6 6   (холодно->горячо: 96 -> 6)
 код (B3F6): значение ×125 (#$7D) -> $4D ; ×100 (#$64) -> $15BF
 роль: база ХХ/обогащения по температуре. Холодно = высоко (96), прогрет = 6.
 ВАЖНО: именно сюда заходит температурный вход — при искажённой ОЖ idle-база не падает.

------------------------------------------------------------------------------
 2) ПРИБАВКА ХХ ПОСЛЕ ПУСКА «A»  @ 0x7EB0  (16 точек)             [ФАКТ]
------------------------------------------------------------------------------
 индекс: $1453 (таймер/состояние; при $1453>=0x41 берётся таблица B, см. ниже)
 сырые:  43 43 35 30 25 19 10 8 7 5 3 1 0 0 0 0     (затухает 43 -> 0)
 код (B5AC): значение ×200 (#$C8) -> $15B6

------------------------------------------------------------------------------
 3) ПРИБАВКА ХХ ПОСЛЕ ПУСКА «B»  @ 0x7F30  (16 точек)             [ФАКТ]
------------------------------------------------------------------------------
 индекс: $1453 (выбирается при $1453>=0x41)
 сырые:  43 43 35 30 25 21 17 11 8 5 3 1 0 0 0 0    (затухает 43 -> 0)
 код (B5AC): ×200 (#$C8)

------------------------------------------------------------------------------
 4) КРИВАЯ по ОБОРОТАМ  @ 0x75A0  (16 точек)                      [ФАКТ]
------------------------------------------------------------------------------
 индекс: $158E (оборото-производный индекс)
 сырые:  255 255 255 255 255 255 153 153 153 153 153 153 153 153 153 153
 роль: ступень высокий/низкий по оборотам (255 на низах -> 153 выше).

------------------------------------------------------------------------------
 5) ОПОРА (TARGET) ЗАМКНУТОГО КОНТУРА ХХ                          [ФАКТ]
------------------------------------------------------------------------------
 формула (B5C9-B5DB): $142C = (175 − $8F) × gain + base
   gain = 0x7F87 = 184      [скаляр XDF]
   base = 0x7F88 = 75       [скаляр XDF]
 $8F = внутренний интегратор контура (НЕ датчик). Ошибка: $144E = 2×$1411 − $142C.


==============================================================================
 ПАРАМЕТРЫ РЕГУЛЯТОРА — ЗАШИТЫ В КОД (не таблица, меняются код-патчем)
==============================================================================
 шаг слежения: #$10=16, #$4B=75 ; лимит шага: #$14=20 ; маска фаз: #$03 (4 фазы)
 драйвер фаз: 4 бита $A7.0x80 / $A7.0x40 / $AB.0x08 / $AB.0x40
              (см. M30_соленоиды_EGR_для_XDF.txt — там разбор фаз шаговика)
 шаг/позиция: routine cpuB4AC (шаг с clamp #$14=20), положение в $1402
 гейт работы контура: $1453 (TST в B413/B430 — при 0 шагание заморожено)

 [ПОПРАВКА 2026-06-23] Раньше тут стояло «драйвер фаз: B4D4 -> порт $1006». ЭТО НЕВЕРНО.
   B4D4 — это рутина ЧТЕНИЯ ADC: вызывается с B=номер канала, пишет (0x20+канал)
   в $1006, ждёт готовности (poll), читает результат из $1007+канал×2, возвращает в D.
   => $1006 — РЕГИСТР КОМАНДЫ ADC, а НЕ порт шаговика; B4D4 — ADC-ридер, НЕ драйвер РХХ.
   Конкретно ADC канал 0 = MAF -> $1408 (тракт MAF см. M30_speed_density_подмена.txt).
   Сам вывод «РХХ шаговый» ОСТАЁТСЯ верным (OCR=0 записей; фазы по битам $A7/$AB,
   шаг/clamp в cpuB4AC). Ошибочной была ТОЛЬКО привязка драйвера к B4D4/$1006.
   Когда нашёл ошибку: при трассировке точки входа MAF — B4D4 явно оказался чтением
   ADC ($1006 команда / $1007 результат), а не выводом GPIO на шаговик.


==============================================================================
 СПИСОК ДЛЯ XDF
==============================================================================
   адрес    тип        назначение                                   индекс
   0x7E20   Table16    Темп. кривая прогрева ХХ                      ОЖ ($B1)
   0x7EB0   Table16    Прибавка ХХ после пуска A                     таймер ($1453)
   0x7F30   Table16    Прибавка ХХ после пуска B                     таймер ($1453)
   0x75A0   Table16    Кривая ХХ по оборотам                         обороты ($158E)
   0x7F87   Constant   Опора контура — gain (184)
   0x7F88   Constant   Опора контура — base (75)


==============================================================================
 ПРИМЕЧАНИЕ про «конкретные цифры в об/мин»
==============================================================================
 Роль, сырые значения, оси-индексы и множители из кода (×125/×100/×200) — ФАКТ.
 НО прямой перевод этих чисел в об/мин невозможен из одной прошивки: РХХ управляет
 ШАГАМИ/положением клапана, а связь «шаг -> об/мин» — физика мотора/клапана, её в ROM
 нет (в отличие от карт топлива/УОЗ, где есть готовые уравнения). Поэтому значения —
 это «единицы регулятора» (шаги/факторы), а не напрямую об/мин.
==============================================================================
