 M30 — ПОЛНЫЙ XDF-ИНВЕНТАРЬ (максимум данных для своей XDF, чисто статически)
 Метод как у Z31: перечислили ВСЕ таблицы, что код читает (LDX#), классиф. по
 lookup-рутине, опознали индексы, проверили значения. CPU = file + 0x8000.
==============================================================================
 Код читает 73 таблицы: 18 в готовом XDF, 55 НОВЫХ.
 Из 55 новых: 39 АКТИВНЫХ, 15 ПЛОСКИХ (выключены/константы — в XDF как скаляр).
 Тип почти всех новых — 1D-кривые (lookup $809F). 2D — $80CF, термистор — $80A9.

 СЛОВАРЬ ИНДЕКСОВ (чем индексируются):
   0x140A/0x140B = ОБОРОТЫ (RPM)            [ДОКАЗАНО]
   0x1482/0x1483 = НАГРУЗКА (Tp)            [ДОКАЗАНО]
   0x1424 = темп. ОЖ ; 0x1489 = темп. воздуха   [ДОКАЗАНО]
   0xB1   = темп-индекс (масштаб 0xCD; роль как 0x8A у Z31)  [ГИПОТЕЗА: воздух]
   0x82   = защёлкнутая копия 0xB1 (темп)
   0x1442, 0x158E = индексы, ВЫЧИСЛЕННЫЕ ИЗ RPM              [ДОКАЗАНО]

==============================================================================
 M30-СПЕЦИФИКА (отличия от Z31)
==============================================================================
 - НЕТ вариантной системы «альт VQ + альт оси» как у Z31 (M30 её не использует).
 - Вместо неё — ПАРА Primary/Knock (детонационная), уже в XDF.
 - НОВЫЙ блок 0x6A00-0x6EFF = таблица УКАЗАТЕЛЕЙ на параметры (Consult/диагностика);
   не «таблица значений», в XDF отдельно — как блок указателей. [структура ДОК.]
 - Гораздо больше коррекционных кривых (55 новых vs 26 у Z31) — больше диагностики/коррекций.


==============================================================================
 ИТОГ ДЛЯ XDF (M30)
==============================================================================
 Готово к записи (структура у всех известна):
   - 39 активных 1D-кривых: ~10 vs температура, ~5 vs обороты, остальные — тип
     известен, вход частично;
   - 15 плоских/констант (пишем как скаляры);
   - + всё из готового XDF (карты Primary/Knock, VQ, оси, тарировки, dwell, cold start);
   - + блок указателей 0x6A00 (Consult).
 Всего ~73 элемента калибровки. Где роль = [ГИПОТЕЗА] — имя рабочее, XDF декодирует верно.
 Потолок статики тот же: точные имена части кривых/датчиков -> нужен мануал той же
 машины или прозвонка ЭБУ.
==============================================================================

   0x7FE0  Low Octane Knock Coeff.              -> 0..8
   0x7B90  Timing retard versus temperature     -> все 0 (таблица есть, выключена)
   0x7F92  O2 Sensor RICH Voltage Limit         -> 0.75 В (X*0.01)
   0x7F93  O2 Sensor LEAN Voltage Limit         -> 0.21 В
   0x7FA4  Rev Limit 2                          -> 5200 об/мин
   0x7FC8  Map Switching Constant               -> 37
   0x7EE0  FLAG: O2 sensor disable
   0x7761  FLAG: Fuel Map Correction Disable
   0x6AD1  ROM ID                               -> данные (5 байт)
   0x7500  Secondary Fuel Map                   -> 5..15 AFR (у M30 правдоподобно)

 НЕ применимо к M30 (RB20-специфика, у нас на этих адресах другое/пусто):
   VTC (0x79F2-0x79FF), 5th-gear/Knock Fuel карты (0x7000/0x7100/0x7200),
   Electric Fan (0x7442-0x7445, 0x7F75/0x7F76), parked rev limiter (0x7FE9).

 ДОПОЛНЕНИЕ: КАЛИБРОВКИ РХХ (холостой ход) — полный разбор в M30_ХХ_РХХ_калибровки.txt
==============================================================================
 РХХ M30 = ШАГОВЫЙ (4 фазы по битам $A7/$AB). Найденные таблицы/скаляры ХХ:
   [ПОПРАВКА] $1006 — это регистр команды ADC (B4D4=ADC-ридер), НЕ порт шаговика;
   см. M30_ХХ_РХХ_калибровки.txt и M30_speed_density_подмена.txt.
   0x7E20  Темп. кривая прогрева ХХ (индекс ОЖ $B1)        96..6
   0x7EB0  Прибавка ХХ после пуска A (индекс $1453)        43..0  (×200)
   0x7F30  Прибавка ХХ после пуска B (индекс $1453)        43..0  (×200)
   0x75A0  Кривая ХХ по оборотам (индекс $158E)            255/153
   0x7F87  Опора контура gain = 184
   0x7F88  Опора контура base = 75   ($142C = (175-$8F)*184+75)
 Параметры регулятора (шаг 16/75, лимит 20, 4 фазы) — ЗАШИТЫ В КОД.
 Значения — в единицах регулятора (шаги/факторы), не в об/мин (см. файл, примечание).
