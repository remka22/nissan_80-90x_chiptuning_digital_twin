==============================================================================
 M30 — ОКОННЫЕ СОЛЕНОИДЫ (EGR-тип) и их КАЛИБРОВКИ для выноса в XDF
 Найдено дизассемблером HD6303. Блок управления ~C040-C520 (cpu).
 СХЕМА КАЖДОГО СОЛЕНОИДА:
   обороты в окне  И  нагрузка в окне  И  доп.условия  И  выдержка(дебаунс)
   -> включить бит выходного байта (соленоид).  Выход из окна -> выключить.
==============================================================================

 ВНИМАНИЕ для XDF: сами выходные байты ($A7/$A9/$AB) — это RAM (выходы), их
 в XDF не пишут. В XDF выносятся КАЛИБРОВКИ — кромки окон и пороги (ROM-байты
 ниже, адрес = смещение в файле = адрес XDF). Они и есть «скаляры вкл/выкл».


==============================================================================
 ТРИ ВЫХОДНЫХ БАЙТА-СОЛЕНОИДА (RAM, пишутся-не-читаются = драйв железа)
==============================================================================
   $00A7  — соленоид-байт #1  (зеркала $A8/$A9-группа, таймеры $16Bx)
   $00A9  — соленоид-байт #2
   $00AB  — соленоид-байт #3
   Каждый несёт несколько битов-соленоидов. EGR-клапан — один из этих битов
   (точный — нужна распиновка ЭБУ; см. конец файла).


==============================================================================
 СВЯЗКА: бит-соленоид  <->  его калибровки (окна/пороги) и переменные-условия
==============================================================================

--- ВЫХОД $00A7 -------------------------------------------------------------
 [соленоид A7.0x04]  set@C0D4
     окно нагрузки($1413): 0x768F=23 .. 0x7690=45
     окно доп($14A2):      0x7790=37 .. 0x7698=80  ; альт 0x76A3=37 .. 0x76A4=80
     переменные: $1413(нагрузка), $14A2
 [соленоид A7.0x10]  set@C275   === КЛАПАН УПРАВЛЕНИЯ МОЩНОСТЬЮ (power valve) ===
     [ФАКТ поведение, подтверждено МАНУАЛОМ стр.880-903]:
       соленоид ВКЛ (клапан ЗАКРЫТ, длинный впуск=момент) при:
         нагрузка($1413) в [0x77A2=32 .. 0x77A3=56]  (= «под большой нагрузкой»)
         И обороты($1460) в [0x77A0=36 .. 0x77A1=64]
       обороты ВЫШЕ 64 -> соленоид СНЯТ -> клапан ОТКРЫТ (короткий впуск=отдача на верхах)
     точка перехода = верхняя граница 0x77A1=64 (~3200 об/мин если $1460≈RPM/50)
     калибр: 0x77A0=36(низ RPM) 0x77A1=64(ВЕРХ=переход) 0x77A2=32(низ нагр) 0x77A3=56(верх нагр) 0x769A=65
     переменные: $1460(RPM), $1413(нагрузка), $1453
 [соленоид A7.0x20]  set@C28E   === power valve, 2-я стадия (через выдержку $152A=50) ===
     калибр: 0x77A0=36 .. 0x77A1=64 ; 0x77A4=50(выдержка)
     переменные: $1460(RPM)
 [соленоид A7.0x40]  set@C139
     пороги: 0x7792=10 ; 0x7794=230   ; переменные: $1402
 [соленоид A7.0x80]  set@C126
     пороги: 0x778D=120 ; 0x7699=0 ; 0x7794=230  ; переменные: $140C, $1402
 сбросы $A7: бит0x04@C0D9, биты0x40/0x80(0xC0)@C181, бит0x10@C26D

--- ВЫХОД $00A9 -------------------------------------------------------------
 [соленоид A9.0x10]  set@C080   <-- EGR (кандидат №1; усилен ПО ИСКЛЮЧЕНИЮ: 2-й оконный
                                    соленоид A7.0x10/0x20 опознан как клапан мощности)
     окно RPM($1460):      0x779E=30 .. 0x7695=40
     окно нагрузки($1413): 0x779D=32 .. 0x7696=45
     порог доп($140C):     0x7697=120
     дебаунс: $1523 до 51 (0x33)
     переменные: $1460(RPM), $1413(нагрузка), $140C
 [соленоид A9.0x40]  set@C032   (условие выше по коду, флаговое)
 [соленоид A9.0x20]  set@BE5F   (условие флаговое)
 [соленоид A9.0x80]  set@BDF5   (условие флаговое)
 сбросы $A9: бит0x80@C41B, бит0x40@C432, бит0x20@C452, бит0x10@C46C

--- ВЫХОД $00AB -------------------------------------------------------------
 [соленоид AB.0x10]  set@A425   <-- НИЗКИЕ обороты ($1460<100) = ХХ/AAC, НЕ EGR
     переменные: $1460(RPM)
 [соленоид AB.0x08]  set@BEF3 и @BF1B   (две ветви — порог/гистерезис)
     пороги: 0x768B=120 ; 0x768D=248 ; 0x768C=100 ; 0x768E=3
     переменные: $140C, $1402
 [соленоид AB.0x40]  set@C158
     пороги: 0x7792=10 ; 0x7795=75 ; 0x7688=5  ; переменные: $1402
 [соленоид AB.0x20]  set@C22B   ; переменные: $1400
 [соленоид AB.0x80]  set@C301   ; самотест (AAAA/55) — вероятно ДИАГНОСТИКА, не соленоид
 сбросы $AB: бит0x80@C4B4, бит0x40@C4CE, бит0x20@C4E8, бит0x10@C508, бит0x08@C51F

==============================================================================
 ПЕРЕМЕННЫЕ-УСЛОВИЯ (RAM, для справки — не в XDF)
==============================================================================
   $1424 = температура ОЖ      (конверсия через таблицу 0x7B30, JSR $80A9)
   $1460 = обороты двигателя    (из 16-бит RPM $140A/$140B)
   $1413 = нагрузка/расход      (вход окна)
   $140C, $14A2, $1402, $1453, $1400 = доп.входы/счётчики условий
   $1523 = дебаунс-счётчик соленоида A9.0x10
   $16Bx = поштучные таймеры соленоидов


==============================================================================
 ОСТАЁТСЯ (честная стена)
==============================================================================
 Было два оконных соленоида ($A7.0x10/0x20 и $A9.0x10). Один ($A7.0x10/0x20)
 опознан как КЛАПАН МОЩНОСТИ (power valve) — подтверждено мануалом по поведению.
 Значит EGR = $A9.0x10 теперь ПО ИСКЛЮЧЕНИЮ ещё увереннее (окно RPM 30..40 + нагрузки
 32..45 + дебаунс — классический профиль EGR, лямбда не участвует).
 Финальную точку (бит -> вывод разъёма -> конкретный соленоид) ставит РАСПИНОВКА/прозвонка.
 Все байты помечены как соленоиды, калибровки вынесены — хватит для раздела XDF
 «Соленоиды (EGR / клапан мощности / эмиссия)» со всеми связками.
==============================================================================



==============================================================================
 ПЕРЕСМОТР: РХХ (регулятор ХХ) = ШАГОВЫЙ. Подтверждено железом + знанием машины.
==============================================================================
 Проверка таймера output-compare (OCR 0x000B/0x000C): у M30 записей в OCR = 0
   -> аппаратный ШИМ НЕ используется -> РХХ M30 НЕ ШИМ, а ШАГОВЫЙ (4 фазы по GPIO).

 РХХ M30 = ШАГОВЫЙ КЛАПАН, 4 фазы = группа «$1402»:
   фазы: бит $A7.0x80, $A7.0x40, $AB.0x08, $AB.0x40
   позиция: $1402 (рейт-лимитед величина, routine cpuB4AC: шаг с clamp #$14,
            пишется из $1527/$0089) — это положение шагового двигателя.
   => 4 бита здесь — НЕ 4 клапана, а ОДИН шаговый актуатор ХХ (4 обмотки/шага).

 ПЕРЕСЧЁТ механизмов M30 (было «15 бит»): реальных устройств ~7-8:
   1) РХХ шаговый (4 бита $1402-группы)        = 1 актуатор
   2) EGR              = $A9.0x10 (окно оборотов 30-40 + нагрузки + дебаунс)
                         [усилен по исключению — см. ниже]
   3) КЛАПАН МОЩНОСТИ (power valve) = $A7.0x10(+0x20 2-я стадия)
                         подтверждено МАНУАЛОМ: ВКЛ (клапан закрыт=момент) на
                         низ/сред оборотах под нагрузкой; ВЫШЕ ~3200 об/мин -> открыт.
                         Переход = 0x77A1=64. [ФАКТ поведение + мануал]
   4) клапан по окну нагрузки       = $A7.0x04
   5) низко-оборотный выход          = $AB.0x10 (реле/idle-up)
   6) диагностика (самотест)         = $AB.0x80  (НЕ устройство)
   7) реле/состояние по флагам $CD/$53 = $A9.0x40/0x20/0x80, $AB.0x20
 M30 — АТМО: буст-соленоида НЕТ (в отличие от Z31).
 Контраст с Z31: там РХХ — ШИМ (OCR), а тут — шаговый. См. ../z31/Z31_соленоиды_для_XDF.txt
